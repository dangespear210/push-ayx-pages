# push-ayx-pages

本仓库用于归档和发布多个独立的 HTML 页面。  
所有页面均为静态内容，不依赖后端服务，可直接通过浏览器访问。

## 项目简介

这是一个通用的 HTML 页面归档仓库，主要用于存放各类独立的静态页面。  
每个页面在仓库中独立存在，互不干扰，方便后续查看、分享或部署。

## 目录说明

```
.
├── pages/          # 存放所有 HTML 页面文件
├── assets/         # 页面引用的资源文件（如 CSS、JS、图片等）
├── index.html      # 仓库首页（可选）
└── README.md       # 本文件
```

各页面文件建议以有意义的名称命名，并放在 `pages/` 目录下，便于管理。

## 页面归档说明

- 所有页面均为纯静态 HTML，无服务器端依赖。
- 页面内可引用同仓库 `assets/` 下的资源。
- 如需新增页面，直接在 `pages/` 下创建 `.html` 文件即可。
- 请确保页面文件路径正确，资源引用使用相对路径。

## 维护说明

- 本仓库仅用于归档，不保证实时更新。
- 提交新页面时，建议附带简要说明（如页面用途、引用来源等）。
- 如发现页面无法正常显示，请检查资源路径或文件编码。
- 欢迎通过 Issue 或 Pull Request 提交改进或新增内容。
